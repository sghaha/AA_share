## 한 줄 요약
소상공인 카페·프랜차이즈를 위한 **AI 기반 모바일 사전주문·결제·픽업 알림·운영 최적화** 서비스. 혼잡 시간대 대기·주문 누락·제조 지연 문제를 해결하는 것이 목적.

## 시스템 구성 (연계 대상)
고객 App · 매장 직원 화면 · 관리자 Web · 외부 결제 · 외부 알림 · **AI 추론 엔진**

- **AI 입력**: 과거 주문 이력, 시간대/요일/날씨, 매장 혼잡도, 메뉴별 판매량, 제조 소요시간, 품절 이력
- **AI 출력**: 예상 주문량, 예상 제조·픽업 시간, 품절 가능성, 주문 지연 가능성
- **처리 흐름 6단계**: 고객/매장 채널 → 매장/메뉴 관리 → 주문/결제 → AI 추론 → 제조/픽업알림 → 운영·AI 대시보드

## 핵심 기능 요구사항 (2장) — 6개 기능군

| # | 기능군 | 핵심 |
|---|---|---|
| 2.1 | 매장·메뉴 정보 관리 | 매장 검색, 메뉴/옵션/가격/품절/운영시간 조회, 관리자 변경(3초 내 조회) |
| 2.2 | 모바일 주문·결제 | 장바구니→주문 생성→**외부 결제 성공 후에만 주문 확정**, 승인/실패/취소 처리 |
| 2.3 | 매장 주문 처리·픽업 알림 | 직원이 접수→제조→픽업 상태 관리, **제조완료 시 Push 알림**(SMS/메신저 확장), 실패 재시도 |
| 2.4 | AI 주문 예측·운영 최적화 | 수요 예측·예상 제조/픽업 시간·품절 가능성·지연 감지 |
| 2.5 | 매장 운영 관리·통계 | 운영시간·주문가능 설정, 주문 일시중지, AI 대시보드 조회 |
| 2.6 | 사용자·보안 관리 | 4개 역할 **RBAC**, 카드정보 미저장, AI용 개인정보 비식별화, 감사 로그 |

### 주문 상태 8종 (상태 머신)
`주문대기 → 결제완료 → 접수완료 → 제조중 → 제조완료 → 픽업완료`
(예외: `주문취소`, `결제실패`)

### AI 기능의 진짜 초점 (2.4 명시)
복잡한 AI 모델 구현이 아니라 → **데이터 수집 · 추론엔진 연계 · 결과 저장 · 관리자 화면 · 알림 연계 · 예외 처리 · 모델 관리 구조 설계**

### AI 경보 4등급
정보(대시보드 표시) → 주의(직원 확인, +30%) → 경고(관리자 알림, +50%/품절위험) → 심각(주문 일시중지 검토, 폭주)

## 시스템 품질요건 (3장) — 아키텍처 드라이버의 원천

| 절 | 품질속성 | 대표 요건 (수치·핵심) |
|---|---|---|
| 3.1 | 성능 | 조회·주문 생성 평균 3초, 상태 조회 2초, 알림 발송 10초, AI 픽업시간 5초 |
| 3.2 | 처리량·동시성 | **매장별 분당 100건**, 전국 500→2,000개 매장, **초기 분당 1만 → 확장 4만 건**, 수평 확장·비동기 큐 |
| 3.3 | 가용성·장애대응 | 결제/알림/AI 장애 시 대체 안내, **AI 장애에도 주문·결제·제조 기능 유지**, Health Check |
| 3.4 | 신뢰성·데이터 일관성 | 폭주에도 **중복 주문·중복 결제·주문 누락 방지**, 상태 변경 이력(시각·변경자·사유) |
| 3.5 | 보안·개인정보 | 전 구간 TLS/HTTPS, 소속 매장 범위 권한, **카드번호 미저장**, AI 데이터 비식별화 |
| 3.6 | AI 신뢰성·모델 관리 | AI는 **참고 정보**(최종 결정은 사람), 오차 모니터링, **버전관리·롤백·룰 대체** |
| 3.7 | 변경용이성·확장성 | 낮은 결합도, 결제수단·알림채널 확장, 관리자 화면에서 코드 수정 없이 변경 |
| 3.8 | 상호운용성·외부 API | 결제/알림/지도/날씨/POS/멤버십 연동, 인증·호출제한·재시도 정책 |
| 3.9 | 감사추적성·백업·복구 | 주문/결제/상태/AI 이력 감사 로그(위변조 방지 별도 저장), 주기적 백업 |
| 3.10 | 사용성·호환성 | 관리자 Web(Chrome/Edge/Safari), App(iOS/Android), 직원 화면(PC/태블릿) |

### ⚠️ 반복 강조되는 최우선 제약 (설계 시 필수 반영)
1. **Graceful Degradation** — 외부 결제·알림·AI가 죽어도 핵심 주문 기능 유지
2. **데이터 일관성** — 폭주에도 중복/누락 없는 주문·결제
3. **AI = 참고 정보** — 룰 대체·버전관리·롤백 가능한 구조
4. **개인정보/결제정보 보호** — 카드정보 미저장, 비식별화, 전 구간 TLS
